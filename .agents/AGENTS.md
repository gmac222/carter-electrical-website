# 🤖 Global AI Agents Guidelines

These guidelines apply universally to all tasks and website development projects in this environment.

---

## ✍️ Language, Grammar & Character Rules

- **UK English**: Always use UK English spelling and grammar throughout all content and documentation (e.g. *optimise*, *colour*, *programme*, *modelling*).
- **Em Dash Ban**: Never use the em dash character (`—`) in any visible copy, page metadata, comments, or headers. Use commas, parentheses, colons, or standard hyphens (`-`) instead.

---

## 🔍 Semantic SEO & Image Naming Rules

When creating or editing content for any landing page or location page, always adhere to the following:
- **Primary Focus**: Human readability must be the primary focus. Content must read naturally, professionally, and clearly as local services pages (not blog posts).
- **Heading Structure & Keyword Distribution**:
  - The single `<h1>` heading must always contain the primary keyword for the page.
  - At least one `<h2>` heading must contain the primary keyword.
  - Other subheadings (`<h2>`, `<h3>`, etc.) must use secondary keywords and location context naturally, avoiding over-optimisation (do not suffix every subheading with the location).
- **Image Filenames and Alt Attributes**:
  - The first main image of the page must be named exactly after the primary keyword (using hyphens, e.g. `electricians-chester.jpg`).
  - The first main image's `alt` attribute must match the primary keyword exactly (e.g. `Electricians in Chester`).
  - Other page images must be named after secondary keywords (e.g. `uploads/commercial-electrical-services-chester.jpg`).
  - Other images' `alt` attributes must match their corresponding secondary keywords exactly.

---

## ❓ Semantic FAQ Rule (People Also Asked)

- **PAA Source of Truth**: Banish generic placeholder or guessed questions. The FAQ section must always be derived from real Google 'People Also Asked' (PAA) questions for the page's target keywords.
- **Data For SEO Integration**: Use the Data For SEO tool (`serp_organic_live_advanced`) to retrieve real-time PAA questions for the page's primary and secondary keywords. If Data For SEO is unavailable, execute a web search query targeting UK PAA search results.
- **Keyword & Client Focus**: Each FAQ question must target either the page's primary keyword or one of the secondary keywords. Answers must represent the client's business directly (e.g. from the contractor's perspective: "We are NICEIC-approved...", "Our local team...") with the primary topic and local intent of the page in mind.
- **Sensitive Pricing Rule**: Never guess or estimate sensitive information such as pricing. If pricing is not explicitly documented elsewhere in the public site content, the agent must ask the user for exact numbers or omit/reframe the pricing FAQ to focus on scope and fixed-quotes.
